# 開発ガイドライン

このプロジェクトへの貢献や開発を行う際のルールとワークフローをまとめています。

## 開発ワークフロー

本プロジェクトでは以下のワークフローを採用しています：

1. **開発ブランチ**: `develop`
   - 日々の開発や機能追加はすべて `develop` ブランチ（または `develop` から派生したトピックブランチ）で行います。
   - ローカル環境で十分に動作確認を行ってください。

2. **リリースフロー**: `main`
   - `main` ブランチはリリース用です。直接のコミットは避け、必ず `develop` からの Pull Request (PR) を通じてマージします。
   - PR作成時に、変更内容のレビューと最終確認を行います。

## その他のルール

- **仕様駆動開発**: 開発は必ず `.specify/specs/` 配下の仕様書を起点としてください。
- **品質保証**: コードの変更後は、必ずビルド (`npm run build`) やテスト (`npm test`) を実行し、動作と型チェックをパスしてからコミットしてください。
