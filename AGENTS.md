# Agents for Baby-App

このドキュメントは、Baby-App プロジェクトにおける AI エージェント（Gemini CLI 等）の役割、利用可能なスキル、および開発ガイドラインについて説明します。

## プロジェクト概要

Baby-App は、育児中の保護者が赤ちゃんのケア記録を管理するための FastAPI ベースの Web アプリケーションです。
バックエンドに FastAPI、フロントエンドに htmx と Tailwind CSS を使用し、データベースとして PostgreSQL（Neon）を利用しています。

## 開発ガイドライン

本プロジェクトでは、以下の原則に従って開発を行います。

### 仕様駆動開発 (Specification-Driven Development)

すべての開発は `.specify/specs/` 配下の仕様書を起点とします。コードを変更する前に、必ず関連する仕様書を確認し、必要に応じて更新してください。

### 検証の徹底

コードの変更を行った後は、必ず以下の手順を実行してください：
1. **ビルドチェック**: `npm run build`（フロントエンド資産がある場合）や環境の整合性確認。
2. **テスト実行**: `pytest` 等によるユニットテストの実行。
3. **型チェック**: `mypy` 等による静的解析。

## 利用可能なスキル

`.agents/skills/` 配下に定義されている特殊なスキルです。

### [neon-postgres](./.agents/skills/neon-postgres/SKILL.md)

Neon Serverless Postgres を操作するためのガイドラインとベストプラクティスを提供します。
- **用途**: データベース接続、ローカル開発、Neon 固有の機能（ブランチング、オートスケーリング等）に関する質問や操作。
- **リソース**: `.agents/skills/neon-postgres/references/` に詳細なリファレンスがあります。

## エージェントへの指示

エージェントは常に日本語で回答し、npm コマンドの実行についてはユーザーへの事前確認を省略して進めることが期待されています。
また、新しい機能の追加やバグ修正の際は、必ずテストコードを含めるようにしてください。
